# Лабораторная работа №1
# Days to New Year Service
Сервис для расчета количества дней до наступления нового года.

### Цель работы
Изучение основ клиент-серверного взаимодействия в веб-приложениях, освоение базовых принципов работы с HTTP-протоколом, получение практических навыков создания простого веб-сервера и обработки HTTP-запросов с использованием различных технологических стеков.

Веб-приложение на **FastAPI**, которое обрабатывает HTTP GET-запросы по маршруту `/` и возвращает JSON с количеством дней до следующего Нового года.

## Функциональность

- Эндпоинт: `GET /`
- Формат ответа: `{"days_before_new_year": <число>}`
- Дата расчёта: от текущего дня до 1 января следующего года
- Язык: Python 3.13
- Фреймворк: FastAPI
- Контейнеризация: Docker / Docker Compose

## Запуск с помощью Docker

### 1. Убедитесь, что Docker установлен и запущен
```bash
docker --version
```
### 2. Соберите Docker-образ
Из корня проекта выполните:
```bash
docker build -t days-to-newyear .
```
### 3. Запустите контейнер
```bash
docker run -p 4200:4200 days-to-newyear
```
### 4. Проверьте работу
Откройте браузер и перейдите по адресу:
http://localhost:4200/
Или выполните запрос через curl:
```bash
curl http://localhost:4200/
```
Пример ответа:

```json
{
    "days_before_new_year": 301
}
```

### Запуск с помощью Docker Compose
Если у вас есть файл docker-compose.yml, выполните в корне проекта:
```bash
docker-compose up
```

Остановка:
```bash
docker-compose down
```
### Локальный запуск (без Docker)
1. Клонируйте репозиторий или перейдите в папку проекта
cd /путь/к/проекту
2. Создайте и активируйте виртуальное окружение
```bash
python -m venv venv
```
Windows:
venv\Scripts\activate

macOS / Linux:
source venv/bin/activate

3. Установите зависимости
```bash
python -m pip install -r requirements.txt
```

4. Запустите сервер
```bash
python -m uvicorn app.main:app --host 0.0.0.0 --port 4200
```

5. Проверьте работу
Перейдите по адресу http://localhost:4200/ в браузере или выполните curl.

## Контрольные вопросы (для защиты лабораторной работы)
Что такое клиент-серверная архитектура и какие ее основные компоненты?
Ответ: Клиент-серверная архитектура — это модель взаимодействия, где клиент (браузер, мобильное приложение) отправляет запросы, а сервер (веб-сервер) обрабатывает их и возвращает ответ. Основные компоненты: клиент, сервер, сеть, протокол обмена данными (например, HTTP).

Какие основные методы запросов поддерживает протокол HTTP? Приведите примеры их использования.
Ответ:
- GET — получение данных (например, GET /)
- POST — создание нового ресурса (например, отправка формы)
- PUT — полное обновление ресурса
- PATCH — частичное обновление ресурса
- DELETE — удаление ресурса

Что такое статус-коды в протоколе HTTP? Приведите примеры наиболее часто используемых кодов.
Ответ: Это трёхзначные числа, которые сообщают клиенту результат обработки запроса.

- 200 OK — успешный запрос
- 201 Created — ресурс создан
- 400 Bad Request — неверный запрос
- 404 Not Found — ресурс не найден
- 500 Internal Server Error — ошибка на сервере

В чем разница между протоколами HTTP и HTTPS?
Ответ: HTTPS — это защищённая версия HTTP. Данные шифруются с помощью SSL/TLS, что предотвращает перехват и подделку информации.

Что такое заголовки HTTP и для чего они используются?
Ответ: Заголовки передают дополнительную информацию о запросе или ответе. Например, Content-Type: application/json указывает формат данных, User-Agent идентифицирует клиента, Authorization передаёт токен для аутентификации.

Что такое контейнеризация и какие преимущества она предоставляет при разработке веб-приложений?
Ответ: Контейнеризация — это упаковка приложения и его зависимостей в изолированный образ. Преимущества: одинаковое поведение на всех средах, простота развёртывания, изоляция, масштабирование.

В чем разница между образами и контейнерами в Docker?

Ответ:
- Образ — это шаблон (набор инструкций и файлов) для создания контейнера.
- Контейнер — это запущенный экземпляр образа (живая работающая программа).

Какие основные типы данных могут передаваться в теле ответа при работе с веб-сервисами?

Ответ:
- JSON (наиболее популярный)
- XML
- HTML
- plain text (текст)
- бинарные данные (изображения, файлы)

Что такое маршрутизация в контексте веб-приложений и как она реализуется?

Ответ: Маршрутизация — это сопоставление входящего URL и метода HTTP с конкретным обработчиком (функцией) в коде. В FastAPI реализуется с помощью декораторов: @app.get("/") привязывает функцию к корневому пути и методу GET.
